# AI Trust Calibration

Открытый эксперимент о калибровке недоверия к современным AI-моделям.

Проект проверяет не тезис «AI не ошибается», а более узкую гипотезу: представление «любой ответ AI нужно перепроверять целиком» может быть плохо откалибровано относительно качества конкретных современных моделей и цены ошибки.

## Что измеряем

1. Субъективную оценку вероятности существенной ошибки до эксперимента.
2. Частоту существенных ошибок в слепо оценённых ответах нескольких моделей.
3. Нагрузку на проверку (`verification burden`).
4. Изменение оценки риска после раскрытия моделей.
5. Связь между привычно используемой моделью и исходной оценкой риска.

## Принцип эксперимента

Участник даёт реальную профессиональную задачу и до генерации фиксирует ожидаемое ядро хорошего ответа, критические ошибки и допустимые спорные места. Backend запускает одну и ту же задачу через несколько AI-провайдеров, случайно назначает ответам метки A/B/C и не раскрывает модели до завершения оценки.

## Статус

Сейчас строится pilot-контур. API-ключи не обязательны для запуска: `Mock` provider позволяет пройти полный пользовательский сценарий. Реальные OpenAI, DeepSeek и Yandex AI Studio провайдеры подключаются конфигурацией после получения ключей.

## Структура

- `docs/research/` — гипотезы, pilot-протокол и rubric оценки.
- `src/AiTrustCalibration.Api/` — ASP.NET Core API и AI provider adapters.
- `web/` — React/Vite интерфейс участника.

## Быстрый запуск

Backend:

```powershell
cd src/AiTrustCalibration.Api
dotnet run
```

Frontend:

```powershell
cd web
npm install
npm run dev
```

По умолчанию включён только `Mock` provider. Секреты не коммитятся; реальные ключи задаются через environment variables или .NET user-secrets.
